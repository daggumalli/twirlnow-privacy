TwirlNow

Privacy Policy

Last updated: 2026-05-07

TwirlNow is a personal automation tool operated by Srikanth Daggumalli. It generates editorial-style carousels and short videos from publicly available AI/tech news headlines and publishes them to the operator's own social channels. This page explains what data the tool handles and how.

Who this policy applies to

This policy applies to anyone who:

  • Lands on this page itself, or
  • Interacts with a TwirlNow-published post on LinkedIn, YouTube, or other social platforms.

Data TwirlNow does NOT collect

TwirlNow is operator-only software. It is not a hosted service. It does not:

  • Collect personal data from people who view or engage with the published posts.
  • Track click-through, IP addresses, or analytics on social engagement.
  • Set cookies or fingerprint browsers.
  • Sell, share, or monetize any user data.

There is no account to sign up for, no email list, and no server-side database tracking individuals.

Third-party services TwirlNow talks to

When the operator runs the tool, the following third-party APIs are queried using credentials the operator owns:

  • News RSS feeds — Google News, Hacker News, VentureBeat, TechCrunch, MIT Tech Review, Wired, Ars Technica, The Verge.
  • X (Twitter) — via the operator's authenticated browser session, to capture publicly visible tweets.
  • Wikimedia Commons — to fetch publicly licensed brand logos for slide rendering.
  • Anthropic / Google Gemini / Groq — for LLM-driven editorial summarization, using API keys the operator provides.
  • LinkedIn API — to publish carousels and videos to the operator's own personal LinkedIn account, using an OAuth token the operator authorized.
  • YouTube Data API — to publish videos to the operator's own YouTube channel.
  • Telegram Bot API — operator-only notifications and pipeline triggers.

Each of these services has its own privacy policy that governs what they do with requests made by TwirlNow on the operator's behalf. TwirlNow does not relay any user data to these services beyond the public content the operator chose to publish.

How OAuth tokens are stored

When the operator authorizes TwirlNow to post on their behalf (LinkedIn, YouTube, etc.), the resulting OAuth tokens are stored as plain-text JSON files in the local config/ directory. These files are listed in the project's .gitignore and are never committed to the public GitHub repository.

Tokens can be revoked at any time:

  • LinkedIn: linkedin.com/psettings/permitted-services → revoke "TwirlNow"
  • YouTube / Google: myaccount.google.com/permissions
  • Or by deleting the local config/*_token.json file.

Source code

TwirlNow is open source. The full implementation lives at github.com/daggumalli/TwirlNow — you can audit exactly what the tool does and how it handles data.

Changes to this policy

If the policy materially changes, the "Last updated" date at the top of this page will be updated. Material changes (e.g., introducing analytics, multi-tenant deployment) will be noted in the repository's commit history.
